Core Basics — Class, Object, Access Modifiers

What is a Class?

A class is a blueprint or template for creating objects. Think of it like a "cookie cutter" — the class is the cutter, and objects are the cookies.

Definition
A class defines attributes (data/variables) and methods (functions/behavior). 
public class Student {
    // ATTRIBUTES (fields/variables)
    String name;
    int age;
    double gpa;

    // METHOD (behavior)
    void study() {
        System.out.println(name + " is studying");
    }
}

To create an object (instance) from this class: 

Student s1 = new Student();  // creating an object
s1.name = "Rahim";           // setting attribute
s1.age = 20;
s1.study();                  // Output: Rahim is studying

Access Modifiers — Who Can See What?

Access modifiers control the visibility of class members.

Modifier Same Class Same Package Subclass Everywhere
private ✅ ❌ ❌ ❌
default (no keyword) ✅ ✅ ❌ ❌
protected ✅ ✅ ✅ ❌
public ✅ ✅ ✅ ✅
private = most restricted. public = open to everyone. protected = open to subclasses. 
public class Person {
    private String name;   // only accessible inside Person
    protected int age;    // accessible in subclasses too
    public String city;   // accessible everywhere
}

static keyword

static means the variable/method belongs to the CLASS itself, not to any specific object. All objects share the same static variable. 

public class StaticCounter {
    static int counter = 0;  // shared by ALL objects

    void incrementCounter() {
        counter++;
    }

    public static void main(String[] args) {
        StaticCounter obj1 = new StaticCounter();
        StaticCounter obj2 = new StaticCounter();
        obj1.incrementCounter();  // counter = 1
        obj2.incrementCounter();  // counter = 2 (same counter!)
        System.out.println("Counter: " + obj1.counter);
        // Output: Counter: 2
    }
}
Yes, you CAN change a static variable's value (unless it's also final). Both obj1 and obj2 share the same counter, so the output is Counter: 2.

final keyword

final means the value CANNOT be changed after assignment. 

final int MAX = 100;
MAX = 200;  // ❌ ERROR! Cannot change a final variable

static final int PI_APPROX = 3;  // constant — shared AND unchangeable
static final = a CONSTANT. Use it for values that never change like PI, MAX_SIZE, etc.

Getters & Setters (Accessors & Mutators)

Since attributes are usually private, we use getters (to read) and setters (to modify) them.

  • Getter (Accessor) — returns the value of a private attribute
  • Setter (Mutator) — sets/changes the value of a private attribute 
public class Person {
    private String name;
    private int age;

    // GETTER (Accessor)
    public String getName() {
        return name;
    }

    // SETTER (Mutator)
    public void setName(String name) {
        this.name = name;  // 'this' refers to current object
    }

    public int getAge() { return age; }
    public void setAge(int age) { this.age = age; }
}

// Using getters & setters:
Person p = new Person();
p.setName("Rahim");
System.out.println(p.getName()); // Output: Rahim
Why use getters & setters?
To protect data. You can add validation inside setters. E.g., reject negative age.

Reference vs Value — A Critical Concept (Q4 in both papers!)

In Java, objects are stored as references (like addresses). When you do obj2 = obj1, BOTH point to the same object! 

// From Fall 2024 Q4a:
String name = "John";
Employee employee1 = new Employee(name);
Employee employee2 = employee1;  // employee2 points to SAME object as employee1!
employee2.setName("Doe");           // Changes the SHARED object
String newName = employee1.getName(); // What is newName?
// Answer: "Doe" — because employee1 and employee2 are the same object!

// From Spring 2025 Q4a:
int age = 25;
Person person1 = new Person(age);
Person person2 = person1;  // SAME reference!
person2.setAge(30);
int newAge = person1.getAge(); // Answer: 30
person2 = person1 does NOT create a copy. They share the same memory. Changes via person2 affect person1!

Constructors — Types & Constructor Overloading
Definition
A constructor is a special method that runs automatically when an object is created. It has the same name as the class and no return type.

3 Types of Constructors

1. Default Constructor
No parameters. Sets default values.
2. Parameterized Constructor
Takes parameters to set values.
3. Copy Constructor
Takes another object and copies its values. 
public class Book {
    String title;
    String author;
    int publicationYear;
    double price;

    // 1. DEFAULT CONSTRUCTOR
    public Book() {
        title = "Unknown";
        author = "Unknown";
        publicationYear = 0;
        price = 0.0;
    }

    // 2. PARAMETERIZED — title and author only
    public Book(String title, String author) {
        this.title = title;
        this.author = author;
        this.publicationYear = 0;
        this.price = 0.0;
    }

    // 3. PARAMETERIZED — title, author, year
    public Book(String title, String author, int year) {
        this.title = title;
        this.author = author;
        this.publicationYear = year;
    }

    // 4. PARAMETERIZED — all four attributes
    public Book(String title, String author, int year, double price) {
        this.title = title;
        this.author = author;
        this.publicationYear = year;
        this.price = price;
    }

    void display() {
        System.out.println(title + " by " + author +
            " (" + publicationYear + ") - $" + price);
    }
}

// Creating objects with different constructors:
Book b1 = new Book();                          // default
Book b2 = new Book("Java", "James");          // 2 params
Book b3 = new Book("OOP", "Grady", 2020, 29.99); // all 4
b1.display(); // Unknown by Unknown (0) - $0.0
b3.display(); // OOP by Grady (2020) - $29.99
Having multiple constructors with different parameters = Constructor Overloading!

The 4 Pillars of OOP

🧱 Encapsulation

Wrapping data (attributes) and methods together in a class. Making attributes private and providing public getters/setters.

🌿 Inheritance

A child class inherits attributes and methods from a parent class. Keyword: extends.

🎭 Polymorphism

One method, many forms. Method Overloading & Overriding. Dynamic Dispatch.

🎨 Abstraction

Hiding implementation details. Using abstract classes and interfaces.

Inheritance — extends keyword

Child class gets all public/protected members of parent. Use super to call parent's constructor/method. 

public class Rectangle {      // PARENT class
    double length;
    double width;

    public Rectangle(double l, double w) {
        length = l; width = w;
    }
    double area() { return length * width; }
    double perimeter() { return 2 * (length + width); }
}

public class Square extends Rectangle {  // CHILD class
    public Square(double side) {
        super(side, side);  // calls Rectangle's constructor
    }

    // OVERRIDES the parent's area method
    @Override
    double area() { return length * length; }

    @Override
    double perimeter() { return 4 * length; }
}

// Usage:
Square sq = new Square(5.0);
System.out.println(sq.area());      // 25.0
System.out.println(sq.perimeter()); // 20.0

Polymorphism — Method Overloading vs Overriding

Overloading (same class)

Same method name, different parameters. Decided at compile time. 

class Calculator {
  int add(int a, int b) {
    return a + b;
  }
  double add(double a, double b) {
    return a + b;
  }
  // Different parameter types = overloading
}

Overriding (parent-child)

Child redefines parent's method. Same name, same parameters. @Override. 

class Animal {
  void sound() {
    System.out.println("Generic sound");
  }
}
class Dog extends Animal {
  @Override
  void sound() {
    System.out.println("Woof!");
  }
}

From Q2a (both papers) — Is this overloading or overriding?

// Fall 2024:
calculateArea(double radius)
calculateArea(int length, int width)
// Answer: OVERLOADING — same class, different parameter types/count

// Spring 2025:
setLength(double length)
setLength(String length)
// Answer: OVERLOADING — same method name, different parameter types
Overloading = same name, different parameters (in SAME class). Overriding = same name, same parameters (in CHILD class).

Dynamic Dispatch, Upcasting & Downcasting

Dynamic Dispatch
Java decides at runtime which version of an overridden method to call, based on the actual object type (not the reference type). 
Animal a = new Dog();  // Upcasting: Dog stored in Animal reference
a.sound();  // Output: "Woof!" — Dog's version is called (dynamic dispatch)

// Downcasting: convert back to child type
Dog d = (Dog) a;  // downcasting
d.fetch();  // now you can call Dog-specific methods
Upcasting = storing child object in parent reference (automatic). Downcasting = extracting back to child type (manual, needs cast).

Abstract Class & Abstract Methods (Q5b in BOTH papers)

Abstract Class
Cannot be instantiated (can't create objects from it). May contain abstract methods (methods without body that subclasses MUST implement). 
abstract class Shape {       // abstract class — keyword: abstract
    String color;

    // Abstract method — NO BODY, subclass must implement it
    abstract double area();

    // Non-abstract method — has a body
    void display() {
        System.out.println("Color: " + color);
    }
}

class Circle extends Shape {   // extends for abstract class
    double radius;

    public Circle(double r) { radius = r; }

    @Override
    double area() { return Math.PI * radius * radius; }
}

class Rectangle extends Shape {
    double length, width;

    public Rectangle(double l, double w) { length = l; width = w; }

    @Override
    double area() { return length * width; }
}

// Shape s = new Shape(); ❌ CANNOT instantiate abstract class
Shape c = new Circle(5.0); // ✅ upcasting
System.out.println(c.area()); // 78.53...

Interface (Q6b in BOTH papers)

Interface
Like a contract. A class that implements an interface MUST provide all its methods. Keyword: implements (not extends!). All methods are abstract by default. 
interface Shape {             // interface keyword
    double area();            // abstract by default (no abstract keyword needed)
    double perimeter();
}

class Circle implements Shape {  // implements keyword!
    double radius;
    Circle(double r) { radius = r; }

    public double area() { return Math.PI * radius * radius; }
    public double perimeter() { return 2 * Math.PI * radius; }
}

class Rectangle implements Shape {
    double length, width;
    Rectangle(double l, double w) { length = l; width = w; }

    public double area() { return length * width; }
    public double perimeter() { return 2 * (length + width); }
}
Feature Abstract Class Interface
Keyword abstract class + extends interface + implements
Methods Can have regular + abstract All abstract (by default)
Variables Any type Only constants (static final)
Multiple inheritance No (only one parent) Yes (can implement multiple)

Polymorphism with LinkedList (Q1b — BOTH papers, most important!) 

import java.util.LinkedList;

// Fall 2024 Q1b — Shape/Rectangle/Triangle/Circle
interface Shape {
    double area();
}

class Rectangle implements Shape {
    double length, width;
    Rectangle(double length, double width) {
        this.length = length; this.width = width;
    }
    Rectangle(double length) { this.length = length; this.width = 0; }
    public double area() { return length * width; }
}

class Triangle implements Shape {
    double base, height;
    Triangle(double base, double height) {
        this.base = base; this.height = height;
    }
    Triangle(double base) { this.base = base; this.height = 0; }
    public double area() { return 0.5 * base * height; }
}

class Circle implements Shape {
    double radius;
    Circle(double radius) { this.radius = radius; }
    Circle() { this.radius = 0; }
    public double area() { return Math.PI * radius * radius; }
}

class Shapes {
    LinkedList<Shape> shapeList;

    public Shapes() {
        // i. Declare the linked list
        shapeList = new LinkedList<>();

        // ii. Create 5 objects: 2 Rectangle, 2 Triangle, 1 Circle
        shapeList.add(new Rectangle(5.0, 3.0));
        shapeList.add(new Rectangle(4.0));         // one param, width = null/0
        shapeList.add(new Triangle(6.0, 4.0));
        shapeList.add(new Triangle(3.0));          // one param, height = 0
        shapeList.add(new Circle(7.0));

        // iii. Access area() for each object
        for (Shape s : shapeList) {
            System.out.println("Area: " + s.area());
        }
    }
}

Exception Handling

What is an Exception?

An exception is an unexpected error that occurs during program execution. Without handling, the program crashes.

Checked Exceptions
Must be handled. E.g., IOException, FileNotFoundException
Unchecked Exceptions
Runtime errors. E.g., NullPointerException, ArrayIndexOutOfBoundsException
Errors
Serious problems. E.g., OutOfMemoryError. Cannot usually be handled.

try-catch-finally Structure

try {
    // Code that might throw an exception
    int[] arr = {10, 20, 30};
    System.out.println(arr[10]);  // ArrayIndexOutOfBoundsException!
}
catch (ArrayIndexOutOfBoundsException e) {
    // Handle the exception here
    System.out.println("Error: Index out of range!");
}
finally {
    // This ALWAYS runs, whether exception occurred or not
    System.out.println("This always runs.");
}

Multiple catch blocks:

try {
    int[] arr = {1, 2, 3};
    int result = arr[0] / 0;  // ArithmeticException (div by zero)
    System.out.println(arr[10]); // ArrayIndexOutOfBoundsException
}
catch (ArithmeticException e) {
    System.out.println("Arithmetic error: " + e.getMessage());
}
catch (ArrayIndexOutOfBoundsException e) {
    System.out.println("Array index out of range!");
}
finally {
    System.out.println("Done.");
}

Q1c Solution — ArrayIndexOutOfBoundsException (BOTH papers)

// The given code had index = 5, but array has indices 0-4 (5 elements)
void arrayIndexException() {
    try {
        int[] myArray = {10, 20, 30, 40, 50};
        int index = 5;  // ❌ Invalid! Valid: 0,1,2,3,4
        int value = myArray[index];
        System.out.println("Value at index " + index + ": " + value);
    }
    catch (ArrayIndexOutOfBoundsException e) {
        System.out.println("ArrayIndexOutOfBoundsException: The specified index is out of range.");
    }
}
// i. With index=5: Exception is caught. Prints the catch message.
// ii. Change index to 3 (valid): Output: "Value at index 3: 40"

// iii. Multiple exceptions including ArithmeticException:
void multipleExceptions() {
    try {
        int[] myArray = {10, 20, 30, 40, 50};
        int result = myArray[2] / 0;  // ArithmeticException
        System.out.println(myArray[10]);   // ArrayIndexOutOfBoundsException
    }
    catch (ArithmeticException e) {
        System.out.println("Cannot divide by zero!");
    }
    catch (ArrayIndexOutOfBoundsException e) {
        System.out.println("Index out of range!");
    }
}

Exception with User Input Loop (Q6c — BOTH papers)

import java.util.Scanner;

public static void multiplyWithExceptionHandling() {
    Scanner scanner = new Scanner(System.in);
    boolean validInput = false;

    while (!validInput) {
        try {
            System.out.print("Enter first number: ");
            int a = Integer.parseInt(scanner.nextLine());

            System.out.print("Enter second number: ");
            int b = Integer.parseInt(scanner.nextLine());

            long result = (long) a * b;

            // Check for overflow
            if (result > Integer.MAX_VALUE) {
                throw new ArithmeticException("Number overflow!");
            }
            System.out.println("Result: " + result);
            validInput = true;  // exit loop
        }
        catch (NumberFormatException e) {
            System.out.println("Invalid input! Please enter a number.");
        }
        catch (ArithmeticException e) {
            System.out.println("Error: " + e.getMessage());
        }
    }
}

Java Collections Framework

List, Set, Map — When to use which?

Collection Ordered? Duplicates? Key-Value? Use When
List (ArrayList, LinkedList) ✅ Yes ✅ Yes ❌ Order matters & duplicates allowed
Set (HashSet) ❌ No ❌ No ❌ Uniqueness required (no duplicates)
Map (HashMap) ❌ No Keys: No, Values: Yes ✅ Store and retrieve by key

Code Examples for List, Set, Map

import java.util.*;

List<String> names = new ArrayList<>();
names.add("Alice");
names.add("Bob");
names.add("Alice");  // duplicate allowed
System.out.println(names); // [Alice, Bob, Alice]

// SET — no duplicates
Set<String> uniqueNames = new HashSet<>();
uniqueNames.add("Alice");
uniqueNames.add("Bob");
uniqueNames.add("Alice");  // ignored! duplicate
System.out.println(uniqueNames); // [Alice, Bob] (order may vary)

// MAP — key-value pairs
Map<String, Integer> scores = new HashMap<>();
scores.put("Alice", 90);
scores.put("Bob", 85);
System.out.println(scores.get("Alice")); // 90

Iterator — What is it? (Q2c — BOTH papers)

Iterator
An object that lets you traverse (go through) a collection one element at a time. Methods: hasNext() — returns true if more elements exist. next() — returns the next element.

Loop approach:

for (String s : names) {
    System.out.println(s);
}
// Simple, but can't remove elements safely

Iterator approach:

Iterator<String> it = names.iterator();
while (it.hasNext()) {
    String s = it.next();
    System.out.println(s);
}
// Can safely remove elements

Enum
A special type that represents a fixed set of constants. Like a list of named values that cannot change. Example: Days of the week, Seasons, Card suits. 
// Simple enum

MVC Theory — Model-View-Controller

MVC Pattern
A design pattern that separates an application into 3 parts to keep code organized and maintainable.
Model (Data)
Contains the data and business logic. No UI code here. Example: Student class with name, grade.
View (UI)
What the user sees. Displays data from Model. Example: JFrame, JPanel, JLabel.
Controller (Logic)
Handles user input. Connects Model and View. Updates Model, tells View to refresh.

Flow: User interacts with View → Controller processes the action → updates Model → View shows updated data.

Example: In a login form — the form is View, login button triggers Controller, user credentials are checked in Model (database).

for (int i = 0; i < arr.length - 1; i++) {

        if (arr[j] > arr[j+1]) {
            int temp = arr[j];     // swap
            arr[j] = arr[j+1];
            arr[j+1] = temp;
        }
    }
}
// arr is now: {1, 2, 3, 5, 8, 9}
System.out.println(Arrays.toString(arr));

Q1a
Access Modifiers + static final 2 marks

Access modifiers control visibility: private (own class only), default (same package), protected (subclasses), public (everywhere).

static final = a constant. Once assigned, it CANNOT be changed. It's shared across all instances. 

public class Demo {
    public static final int MAX_SIZE = 100;

    public static void main(String[] args) {
        System.out.println(MAX_SIZE); // 100 — can access
        // MAX_SIZE = 200; ❌ ERROR: cannot change final
    }
}

Justification: static means it belongs to the class (not instances), so accessible via ClassName.MAX_SIZE or directly. final prevents reassignment — any attempt causes a compile-time error.

Spring 2025 Q1a also asks: Can a static method be accessed by an object? — YES: obj.staticMethod() works, but it's better practice to use ClassName.staticMethod().

import java.util.LinkedList;

abstract class Animal {
    abstract void noise();  // abstract method
}

class Cow extends Animal {
    String sound;
    Cow(String sound) { this.sound = sound; }
    Cow() { this.sound = null; }   // parameter may be null
    void noise() {
        System.out.println(sound != null ? sound : "Moo");
    }
}

class Sheep extends Animal {
    String sound;
    Sheep(String sound) { this.sound = sound; }
    Sheep() { this.sound = null; }
    void noise() {
        System.out.println(sound != null ? sound : "Baa");
    }
}

class Animals {
    LinkedList<Animal> animalList = new LinkedList<>();

    public Animals() {
        // 3 Cow, 2 Sheep (Spring 2025)
        animalList.add(new Cow("Moooo"));
        animalList.add(new Cow("Muuu"));
        animalList.add(new Cow());            // null parameter
        animalList.add(new Sheep("Baaa"));
        animalList.add(new Sheep());          // null parameter

        for (Animal a : animalList) {
            a.noise();  // dynamic dispatch!
        }
    }
}

Reference Behavior — newName / newAge value 2 marks

Fall 2024: employee2 = employee1 → both reference the SAME object. employee2.setName("Doe") changes the shared object. So employee1.getName() returns "Doe".

Spring 2025: person2 = person1 → same reference. person2.setAge(30) changes shared object. So person1.getAge() returns 30.

Step-by-step justification: (1) person1 created with age=25. (2) person2 = person1 means person2 is not a copy — it's an alias to the same object. (3) person2.setAge(30) modifies the one shared object. (4) person1.getAge() returns 30.

Q6a
Accessor & Mutator — when to use them 2 marks

We use accessors (getters) and mutators (setters) when class attributes are private (to protect data).

Accessor: Used when we need to read the value of a private attribute from outside the class. Example: getName()

Mutator: Used when we need to change the value of a private attribute from outside the class. We can add validation inside. Example: setAge() can check if age is positive. 

class BankAccount {
    private double balance;

    // Accessor
    public double getBalance() { return balance; }

    // Mutator with validation
    public void setBalance(double amount) {
        if (amount >= 0) balance = amount;
        else System.out.println("Balance cannot be negative!");
    }
}

Common Mistakes to Avoid

Using extends for interface. Correct: implements for interface, extends for class/abstract class.
Forgetting @Override when overriding — it's not mandatory but expected in exams.
Confusing overloading (same class, different params) with overriding (child class, same params).
Thinking obj2 = obj1 creates a copy. It doesn't — both point to the SAME object!
Forgetting that abstract classes CANNOT be instantiated with new.
Missing public on interface method implementations — they must be public!

Q4 — 4 marks

Write a Java program that reads an integer from user input and handles NumberFormatException. Use a finally block. Keep asking until valid input is provided. 

import java.util.Scanner;
public class InputHandler {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        boolean valid = false;
        while (!valid) {
            try {
                System.out.print("Enter an integer: ");
                int num = Integer.parseInt(sc.nextLine());
                System.out.println("You entered: " + num);
                valid = true;
            } catch (NumberFormatException e) {
                System.out.println("Invalid! Not a number.");
            } finally {
                System.out.println("Attempt complete.");
            }
        }
    }
}
Q5 — 1+3 marks

Why do we use enum in Java? Write an enum called Season with values SPRING, SUMMER, AUTUMN, WINTER, each having a description attribute. Print each season with its description.

Why enum? Enums represent a fixed set of named constants. They make code more readable, type-safe, and prevent invalid values.

public enum Season {
    SPRING("Warm and blooming"),
    SUMMER("Hot and sunny"),
    AUTUMN("Cool and colorful"),
    WINTER("Cold and snowy");

    private final String description;

    private Season(String desc) {
        this.description = desc;
    }

    public String getDescription() { return description; }
}

// Print all:
for (Season s : Season.values()) {
    System.out.println(s + ": " + s.getDescription());
}
